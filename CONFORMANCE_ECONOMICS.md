# Conformance Economics

Aurora and Platinum conformance work should track both quality movement and the
local compute/model effort needed to create it.

The current v1 artifact is generated by:

```sh
npm run harness:analyze:conformance-economics
```

The latest report and charts are under:

```text
reference-artifacts/analyses/conformance-economics/
```

## What It Measures

- score trends across existing conformance reports
- largest metric deltas by axis
- artifact volume as a historical compute proxy
- measured wall time, CPU time, and artifact-byte growth for wrapped future runs
- declared resource classes such as `cpu`, `browser`, `video`, `gpu`, `model-api`,
  and `openai-api`

## How To Measure Future Work

Wrap expensive or meaningful commands with:

```sh
npm run harness:measure -- \
  --axis level-arc \
  --resource cpu \
  --resource browser \
  --resource video \
  --notes "Stage 4 pressure replay sweep" \
  -- npm run harness:check:stage4-pressure-loss-windows
```

When a run uses model or API assistance, declare it without logging secrets or
raw prompts:

```sh
npm run harness:measure -- \
  --axis audio \
  --resource cpu \
  --resource model-api \
  --resource openai-api \
  --model-provider openai \
  --model gpt-5.5 \
  --model-calls 3 \
  --input-tokens 42000 \
  --output-tokens 9000 \
  --notes "Model-assisted cue-window clustering plan" \
  -- npm run harness:analyze:platinum-audio-conformance-lab
```

For local GPU work, start with declared tracking:

```sh
npm run harness:measure -- \
  --axis graphics \
  --resource gpu \
  --gpu-mode local-metal-or-browser \
  --notes "Frame/pixel graphics comparison batch" \
  -- npm run harness:check:galaxy-guardians-frame-motion-conformance
```

True macOS GPU/energy counters may require privileged samplers such as
`powermetrics`; keep those optional and explicit.

## Planning Rule

Each major phase should report:

- score movement by axis
- whether the gain was gameplay-facing, measurement-facing, or both
- wall/CPU/GPU/API effort where available
- artifact volume and retry count
- next highest-ROI target

This keeps long-cycle conformance work from becoming only "more compute." The
goal is better evidence, better game feel, and reusable harness logic per unit
of local resource spent.
